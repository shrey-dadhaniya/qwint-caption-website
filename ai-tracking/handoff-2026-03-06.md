### 2. Success page stuck at "Preparing your plugin"

**Root cause (double bug):**

**Bug A — `check-key` didn't attempt recovery on `processing` state**
- `fulfillmentByOrder` is in-memory. After a server restart, state is loaded from persisted file, but the async promise that was running `processPaidFulfillment` is gone.
- State persisted as `processing` but the work never completed.
- `/api/check-key` only triggered `recoverAndQueueFulfillment` if state was missing or `awaiting_payment` — NOT when stuck in `processing`.
- Fixed: recovery now also fires when state is `processing`.

**Bug B — `enqueueFulfillmentForPayment` didn't try artifact restore on `processing` state**
- When the same payment was already in `processing`, it returned early without checking Razorpay customer notes.
- If fulfillment actually succeeded previously (key+zip URL in Razorpay notes) but server restarted mid-run, state was irrecoverable.
- Fixed: now calls `restoreReadyStateFromArtifacts` when state is `processing` before returning.

**Bug C — Poll loop (UX) had no timeout**
- Fixed: after 28 attempts (~70s), poll gives up and shows a support message instead of spinning forever.
