## Known Issues
- Fulfillment status is currently tracked in-memory (`Map`) in `src/server.js`. If the process restarts during fulfillment, in-flight order state is lost.
- Legacy fallback to `metadata.stripe_id` is retained for existing older keys.
- Razorpay customer note updates are best-effort; if Razorpay rejects `notes` on update, payment fulfillment still succeeds and logs the sync failure.
- New landing template currently uses placeholder behavior for demo/download CTA actions (intended during UI-only phase).
- UI verification has been compile-level (EJS render + syntax checks); visual/browser QA pass is still pending.
